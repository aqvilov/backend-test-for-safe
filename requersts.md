## Запросы к backend для витрины приложений

**Базовый URL (пример)**: `https://your-api.com`  
Все пути ниже указаны относительно базового URL (т.е. полный адрес будет `https://your-api.com/api/...`).

### Основные запросы каталога приложений (React‑фронтенд)

| Метод | URL                         | Что делает                               | Используется во фронте |
|-------|-----------------------------|------------------------------------------|-------------------------|
| GET   | `/api/apps`                 | Получить список всех приложений          | общие списки, поиск по умолчанию |
| GET   | `/api/apps/:id`            | Получить детали конкретного приложения   | страница приложения (AppDetails) |
| GET   | `/api/apps?featured=true`   | Получить список рекомендованных приложений | главная страница (блок «Рекомендуемые») |
| GET   | `/api/apps?topWeek=true`    | Получить топ приложений за неделю        | главная страница (блок «Топ недели») |
| GET   | `/api/apps?search=:query`   | Поиск приложений по названию/категории   | страница поиска (Search) |
| GET   | `/api/categories`           | Получить список категорий                | главная, страница категорий |
| GET   | `/api/categories/:id/apps`  | Получить приложения в выбранной категории | страница категорий |

### Запросы, связанные с пользователем (профиль)

| Метод | URL                     | Что делает                                | Используется во фронте |
|-------|-------------------------|-------------------------------------------|-------------------------|
| GET   | `/api/user/downloads`   | Список скачанных пользователем приложений | профиль, вкладка «Скачанные» |
| GET   | `/api/user/favorites`   | Список избранных приложений               | профиль, вкладка «Избранное» |
| GET   | `/api/user/reviews`     | Список отзывов пользователя               | профиль, вкладка «Отзывы» |

### Запросы по отзывам для приложения

| Метод | URL                      | Что делает                                | Используется во фронте |
|-------|--------------------------|-------------------------------------------|-------------------------|
| GET   | `/api/apps/:id/reviews` | Получить список отзывов для приложения    | страница приложения (вкладка «Отзывы») |
| POST  | `/api/apps/:id/reviews` | Создать новый отзыв для приложения        | для кнопки «Написать отзыв» (нужно подключить) |

## Запросы сервиса отзывов (Flask backend в папке `reviews`)

Эти эндпоинты уже реализованы во Flask‑сервисе и могут использоваться фронтендом.

### Служебный

| Метод | URL           | Что делает                         |
|-------|---------------|------------------------------------|
| GET   | `/api/health` | Проверка доступности сервиса и БД |

### Отзывы и рейтинг

| Метод | URL                              | Что делает                                                      |
|-------|----------------------------------|-----------------------------------------------------------------|
| GET   | `/api/apps/<int:app_id>/reviews` | Получить отзывы по `app_id` (используется вместо mock `getReviewsForApp`) |
| POST  | `/api/apps/<int:app_id>/reviews` | Создать новый отзыв для приложения                              |
| POST  | `/api/reviews/<int:review_id>/like` | Поставить лайк отзыву (кнопка «Полезно» на фронте)           |
| GET   | `/api/apps/<int:app_id>/rating`  | Получить средний рейтинг и количество отзывов для приложения   |

---

## Как использовать этот список

- **Для фронтенда**: в `src/services/api.js` заменить mock‑функции на реальные `fetch`/`axios` вызовы по этим URL.
- **Для бэкенда**: реализовать все указанные эндпоинты с такой же структурой ответов: `{ success: true, data }` или `{ success: false, error }` для основных API каталога, и совместимыми JSON‑структурами для сервиса отзывов.
